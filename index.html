03. Featured Project

Built to Solve Real Problems

Case Study

Healthcare Management System

A full-stack data management platform designed to modernize healthcare record-keeping and operational workflows.

The Problem

Healthcare facilities operating on paper-based or fragmented digital systems face critical inefficiencies: patient records are siloed across departments, appointment scheduling is manual and error-prone, and there is no centralized mechanism to query, audit, or report on operational data. This results in delayed diagnoses, duplicated patient entries, and zero visibility into facility performance metrics — all at the direct expense of patient outcomes.

The Solution

I designed and developed a fully functional, web-based Healthcare Management System that centralizes all patient and operational data behind a secure, role-based interface. The system enables healthcare workers to register patients, manage consultation records, schedule appointments, and generate on-demand reports — all from a single dashboard. Data integrity is enforced through a normalized relational database, while the Flask backend exposes clean routes for all CRUD operations, making the system extensible for future API integrations or cloud deployment on AWS.

Technical Stack

Python Flask SQL HTML / CSS Relational DB Design REST Architecture
  • Backend: Python + Flask — lightweight, scalable, production-ready web framework.
  • Data Layer: SQL with normalized schema design — enforcing referential integrity across patient, appointment, and clinical record tables.
  • Frontend: Semantic HTML5 + CSS3 — clean, responsive UI designed for clinical staff with minimal technical background.
  • Architecture: MVC pattern with modular route handling, enabling straightforward feature extension and testability.
